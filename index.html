jQuery Events

Why Events?

  • Events are what let jQuery really fly. They are what let your users fully appreciate having Javascript enabled on the page
  • Events allow us to easily manipulate DOM nodes after the user does some interaction, such as hovering, clicking and tabbing into form fields
  • Eventing is pretty simple to implement, as per usual, it just requires one more jQuery function

Making sure you target elements properly

Finally, let's dial it back a bit and make sure we select elements properly

For this example let's make a list

  • Hi I'm item #1!
  • Hi I'm item #2 I'm cool too!
  • What up I'm slacker item #3!

First let's attach an event listener to the list, on click each list item should turn red.

We might (erroneously) begin by targeting it in the first way (see the JS)

But this way has a problem! Open up the page in debug mode and in web inspector, add a new list item with the same class

No matter how much we click that one, it won't turn red!

This is because click handlers are attached when the page loads, and this node was added after the page loads so it has no handler attached!!

So let's show a second, better click handler definition to account for this!

This is called Event Delegation it's an important concept, but the tl;dr read of it is:

If you know the element you want to attach an event too might end up having more of it's kind put on the page via Javasript, then attach the event to it's parent, and then in the dot on function, specify what child of the parent must be clicked for the event to run

Summing up the dot on

if we look now, we've used every input the dot on function has in its documentation

.on("event", "child selector this must be clicked/hovered/etc for event to run (this is optional)", {"event.data" : "this is also optional"}, theFunctionToRun);
